# データモデル仕様

## 概要

ライブラリは **Polygon**（ジオメトリを持つオブジェクト）を管理する。
階層・グループの概念はアプリ層に委譲し、ライブラリはジオメトリ操作に徹する。

---

## Polygon（ポリゴン）

地図上の1つの閉じた領域を表す。ジオメトリを直接所有する第一級オブジェクト。

```
Polygon {
  id           : PolygonID          // 一意識別子（自動生成・変更不可）
  geometry     : GeoJSON Polygon    // 単一ポリゴン（interior ring によるドーナツ形を含む）
  display_name : string             // 表示名（任意・空可）
  metadata?    : { [key: string]: any }
  created_at   : datetime
  updated_at   : datetime
}
```

### ID の仕様

- 生成時に自動付与されるハッシュ文字列（例：UUID v4）
- **変更不可**。システム全体でポリゴンを一意に識別する
- 外部データのインポート時も ID はライブラリが採番する（元データの ID は `metadata` に格納して保持する）

### display_name の仕様

- **任意項目**。空でもよい
- ユーザーが自由に編集・変更できる
- 分割・生成直後はデフォルトで空
- 後からいつでも変更可能

### geometry の仕様

- 型は常に **GeoJSON Polygon**（`MultiPolygon` は使用しない）
- 飛び地は複数の Polygon で表現する（アプリ層でグルーピング）
- ドーナツ形（穴あき）は GeoJSON の interior ring で表現する

## 外輪郭キャッシュ（Union Cache）

複数ポリゴンの外輪郭（union）を計算・キャッシュする仕組み。インメモリのみで動作する。

### 基本キャッシュ

```
computeUnion(polygonIds: PolygonID[]) → UnionCacheID
```

指定ポリゴン群の union を計算し、キャッシュに登録する。

### 階層キャッシュ（カスケーディング）

```
computeUnionFromCaches(cacheIds: UnionCacheID[]) → UnionCacheID
```

既存キャッシュの結果を組み合わせて union を算出し、新しいキャッシュとして登録する。
多段階の階層に対応する（例: 丁目 → 区 → 市区全体）。

### キャッシュ取得・削除

```
getCachedUnion(cacheId: UnionCacheID) → GeoJSONPolygon[] | null
deleteCachedUnion(cacheId: UnionCacheID) → void
```

### 自動 dirty 化とカスケーディング伝播

ソースポリゴンが変更・削除されると、関連するキャッシュが自動的に dirty になる。
dirty 伝播はカスケーディング: リーフキャッシュ → 中間キャッシュ → トップキャッシュへと伝播する。
`getCachedUnion` 呼び出し時に dirty なら遅延再計算される。

## 共有境界（Shared Edge）

### 概念

同座標の頂点を共有するポリゴン間で、頂点を移動すると**両方のポリゴンが同時に更新される**。

### データ設計方針：重複方式（各ポリゴン独立保持）

各 Polygon は**完全な自己完結型 GeoJSON geometry** を持つ。
共有頂点は各ポリゴンが独立して保持し、「同座標である」という事実によって共有を表現する。

```
// 共有辺 C-D を持つ2ポリゴン
Polygon1.geometry.coordinates = [[A, B, C, D, A]]   // C と D を独自に保持
Polygon2.geometry.coordinates = [[D, C, E, F, D]]   // C と D を独自に保持（逆順）
```

### 連動対象の範囲

`sharedEdgeMove` の影響は**全ポリゴン**に及ぶ。同座標（epsilon 以内）の頂点を持つ全ポリゴンが連動更新される。

| ケース | 連動するポリゴン |
|--------|----------------|
| 通常の隣接境界 | 同座標の頂点を持つ全ポリゴン |
| ドーナツ＋内側（バチカン型） | 穴リングも検索対象 |
| 3ポリゴンが1点で接する角 | 同座標の全ポリゴン |

### 共有頂点の判定

2頂点が同座標（epsilon 以内）にある場合、共有頂点とみなす。
epsilon は設定可能とする（デフォルト: 1e-8 度 ≒ 1mm 精度）。

## ドラフト接続モデル（Draft Connectivity）

### 概念

PersistedDraft はポリゴン頂点間を結ぶポリラインとして機能する。
複数のドラフトとポリゴン境界を介して**閉回路**が形成されると、
その領域を新しいポリゴンとして自動生成できる。

### 接続グラフ

ドラフト端点がポリゴン頂点と同座標（epsilon 以内）にある場合、
そのドラフトはポリゴンに**接続**しているとみなす。

```
Polygon A ──(draft D1)──> Polygon B ──(draft D2)──> Polygon C
    ↑                                                    │
    └─────────────────(新しい描画)──────────────────────────┘
                         = 閉回路 → 新ポリゴン生成
```

### ドラフト端点インデックス

ドラフト端点の空間検索を効率化するため、ライブラリ内部で
ドラフト端点のハッシュインデックス（`draftEndpointIndex`）を保持する。

- ポリゴン頂点の `coordIndex` と同じ量子化グリッドを使用
- ドラフトの保存・削除時に増分更新

## ワインディング方向（Winding Order）

| 方向 | 意味 |
|------|------|
| 反時計回り（CCW） | **外輪郭の標準**。GeoJSON RFC 7946 準拠 |
| 時計回り（CW） | 穴（ホール）の標準 |

ライブラリは内部で方向を正規化し、ユーザーがどちら向きに頂点を入力しても正しく処理する。
