# bridgePolygons — 2つのポリゴンの隙間を埋める

## この機能が解決する問題

地図上で区画を管理していると、隣接するポリゴン同士の間に**隙間**が生じることがある。
道路を挟んだ2つの区画の間、あるいは不整形な区画の間の空白地帯を
新しいポリゴンとして埋めたい。

```
  ┌────┐         ┌────┐
  │    │         │    │
  │ A  │  隙間   │ B  │     ← この隙間を新しいポリゴンで埋めたい
  │    │         │    │
  └────┘         └────┘
```

`bridgePolygons` は、A の頂点から B の頂点へ線を引くことで、
ポリゴン境界と描画線で囲まれた閉領域を新しいポリゴンとして自動生成する。

---

## 類似 API との違い

| | `bridgePolygons` | `expandWithPolygon` |
|---|---|---|
| **対象** | **2つ**の既存ポリゴンの間 | **1つ**のポリゴンの外側 |
| **元ポリゴンへの影響** | **不変**（一切変更しない） | **削除**される |
| **ユーザ操作** | A上の頂点 → 外部パス → B上の頂点 | 境界上の点A → 外部パス → 境界上の点B |

---

## ユースケースとアプリ側の呼び出しフロー

### 想定シーン

ユーザが A の頂点をタップし、外部を通って B の頂点まで線を引く。
共有辺がある場合は即座にポリゴンが完成し、ない場合は既存ドラフトの蓄積で
段階的にループが閉じる。

### 呼び出し例

```typescript
const result = await editor.bridgePolygons(
  polygonAId,
  aVertexIndex,   // A 上の開始頂点インデックス
  polygonBId,
  bVertexIndex,   // B 上の着地頂点インデックス
  bridgePath,     // ユーザが描いた外部パス
  "隙間ポリゴン",
);

if (result.ok) {
  // ポリゴン生成成功 — 地図に表示
  renderPolygon(result.polygon);
} else {
  // 閉じられなかった — ドラフトとして保存済み
  // 次回の bridgePolygons で閉回路が成立する可能性がある
  showMessage("線を保存しました。さらに線を追加するとポリゴンが完成します");
}
```

## API

```typescript
type BridgeResult =
  | { ok: true;  polygon: MapPolygon }       // 成功
  | { ok: false; draft: PersistedDraft };    // 未完成 → ドラフト保存

bridgePolygons(
  polygonAId   : PolygonID,
  aVertexIndex : number,
  polygonBId   : PolygonID,
  bVertexIndex : number,
  bridgePath   : Point[],       // A頂点 → 外部 → B頂点
  name         : string
): Promise<BridgeResult>
```

- A, B のポリゴンは**一切変更されない**
- 成功時: 新しいポリゴンが追加される
- 失敗時: パスが `PersistedDraft` として保存される

---

## 3段階の判定フロー

ブリッジは以下の順序で判定し、最初に成功した段階で結果を返す。

### 段階1: 直接ブリッジ（A と B が共有辺を持つ場合）

```
  ┌────┬────┐                  ┌────┬────────┬────┐
  │    │    │                  │    │ bridge │    │
  │ A  │ B  │   + 外部パス →  │ A  │ (新規) │ B  │
  │    │    │                  │    │        │    │
  └────┴────┘                  └────┴────────┴────┘
  共有辺あり
```

ブリッジパス + 各ポリゴンの境界ウォーク + 共有辺で閉じたリングを構成する。

### 段階2: 閉回路検出（既存ドラフトを経由してループが閉じる場合）

```
  ┌──┐        ┌──┐        ┌──┐
  │A │        │B │        │C │
  │  ├─ D1 ──┤  ├─ D2 ──┤  │
  └──┘        └──┘        └──┘

  1回目: A→B  → 閉じない → D1保存
  2回目: B→C  → 閉じない → D2保存
  3回目: C→A  → D1+D2+今回の線でループ成立 → ポリゴン生成!
                D1, D2 は自動削除
```

過去に保存されたドラフト（`PersistedDraft`）とポリゴン境界を介して
閉じたループが形成されるかを BFS で検出する。

### 段階3: ドラフト保存（ループが閉じない場合）

```
  ┌──┐        ┌──┐
  │A ├── ? ──┤B │      → パスを PersistedDraft として保存
  └──┘        └──┘        次回以降の bridgePolygons で使われる
```

---

## エッジケースと挙動

| ケース | 挙動 |
|--------|------|
| A と B が共有辺を持つ | 段階1: 即座にポリゴン生成 |
| 共有辺なし＋既存ドラフトでループ成立 | 段階2: ポリゴン生成 + 消費ドラフト削除 |
| 共有辺なし＋ループ不成立 | 段階3: ドラフト保存。`{ ok: false }` を返す |
| 同一ポリゴンの2頂点を指定 | `expandWithPolygon` 等を使うこと |
| 始点=終点のドラフト | 閉回路検出グラフから除外される |

---

## 制約・注意事項

| 項目 | 内容 |
|------|------|
| **元ポリゴンの不変性** | A, B は一切変更されない |
| **coordIndex** | 新ポリゴンの頂点は登録され、`sharedEdgeMove` で連動する |
| **ドラフト消費** | 閉回路検出で使用されたドラフトはストレージから自動削除 |
| **Undo** | 新ポリゴンのみ削除。消費済みドラフトの復元は Undo 対象外 |
| **ワインディング** | 結果リングは CCW に正規化される |
