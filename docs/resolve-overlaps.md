# resolveOverlaps — 重複ポリゴンの非重複分割

## この機能が解決する問題

地図上でポリゴンを描いていると、意図せず（あるいは意図的に）ポリゴン同士が重複することがある。
「2つの営業エリアが重なっている」「隣接する区画の境界が曖昧で一部重複している」といったケースで、
重複領域を独立したポリゴンとして切り出し、各ポリゴンを非重複にしたい。

```
  入力:                              結果:
  ┌──────────┐                      ┌──────┬───┬──────┐
  │    A     │                      │      │   │      │
  │     ┌────┼────┐           →     │ A残  │ AB│ B残  │
  │     │ 重複│    │                │      │   │      │
  └─────┼────┘    │                └──────┴───┴──────┘
        │    B    │                  3つの非重複ポリゴン
        └─────────┘                  A残: ID=A（維持）
                                     B残: ID=B（維持）
                                     AB:  新規ID
```

---

## 類似 API との違い

| | `resolveOverlaps` | `resolveOverlapsWithDraft` |
|---|---|---|
| **入力** | 2つ以上の**確定済み**ポリゴン | 1つの確定済みポリゴン + **未確定ドラフト** |
| **タイミング** | ポリゴン作成後に事後的に呼び出す | ドラフト描画中にリアルタイムで呼び出す |
| **結果** | 全重複パターンを分解（べき集合分解） | ドラフト線の交錯部のみを切り出す |

| | `resolveOverlaps` | `splitPolygon` |
|---|---|---|
| **対象** | **複数**のポリゴンの重複 | **1つ**のポリゴンを線で分割 |
| **元ポリゴンの ID** | **維持**される | **削除**される |

---

## ユースケースとアプリ側の呼び出しフロー

### 想定シーン

ユーザが2つのポリゴンを描いた後、「重複部分を独立したエリアにしたい」と操作する。
あるいは、インポートしたポリゴンデータに重複があり、一括で整理したい場合。

### 呼び出し例

```typescript
// ユーザが地図上で2つの重複ポリゴンを選択した
const selectedIds = [polygonAId, polygonBId];

const result = await editor.resolveOverlaps(selectedIds);

// result.modified: 縮小された A と B（元の ID を維持）
// result.created:  重複領域の新ポリゴン

// アプリ側で地図表示を更新
for (const p of result.modified) {
  updatePolygonOnMap(p);
}
for (const p of result.created) {
  addPolygonToMap(p);
}
```

## API

```typescript
resolveOverlaps(
  polygonIds: PolygonID[]     // 2つ以上
): Promise<{
  modified: MapPolygon[],     // 縮小された元ポリゴン（ID維持）
  created: MapPolygon[],      // 新規の交差領域ポリゴン
}>
```

## 動作の図解

### 2ポリゴンの場合

```
入力: A（面積4）, B（面積4）, 重複領域（面積2）

  A ──────┐
  │       │
  │   ┌───┼── B
  │   │ AB│  │
  └───┼───┘  │
      │      │
      └──────┘

結果:
  modified: [A残（面積2, ID=A）, B残（面積2, ID=B）]
  created:  [AB（面積2, 新規ID）]

  面積の合計: 2 + 2 + 2 = 6 = 元の A(4) + B(4) - 重複(2)
```

### 3ポリゴンの場合（べき集合分解）

```
    A ───────┐
    │  ┌─────┼─── B
    │  │  ┌──┼──┐ │
    └──┼──┼──┘  │ │
       │  │  C  │ │
       └──┼─────┘ │
          └───────┘

結果: 最大7つの領域（各ペア重複 + トリプル重複 + 各排他領域）
  - A のみ
  - B のみ
  - C のみ
  - A∩B（Cを除く）
  - A∩C（Bを除く）
  - B∩C（Aを除く）
  - A∩B∩C
```

N 個のポリゴンに対し、ビットマスクで全部分集合（2^N）を列挙する。
実用上は 2〜5 ポリゴンを想定。

### 重複なしの場合

ポリゴン間に重複がなければ `{ modified: [], created: [] }` を返し、何も変更しない。

---

## アルゴリズム概要

1. **排他領域の計算**: 各ポリゴンから他の全ポリゴンの union を差し引く
2. **交差領域の計算**: サイズ≥2の部分集合ごとに intersection を計算し、含まれないポリゴンの union を差し引く
3. **面積チェック**: 微小スライバー（< 1e-12）を破棄
4. **元ポリゴンの更新**: 排他領域で geometry を更新（ID 維持）
5. **交差ポリゴンの追加**: 新規 ID で追加

---

## エッジケースと挙動

| ケース | 挙動 |
|--------|------|
| 重複なし | `{ modified: [], created: [] }` を返す。変更なし |
| 完全包含（A が B を完全に含む） | A残 = A-B（ドーナツ型）、交差部 = B全体、B残 = 空 |
| 微小スライバー | 面積 < 1e-12 で自動破棄 |
| 重複 ID（同じ ID を2回指定） | 自動重複排除 |
| 3つ以上のポリゴン | べき集合分解で全パターンを処理 |

---

## 制約・注意事項

| 項目 | 内容 |
|------|------|
| **元ポリゴンの ID** | 維持される（geometry のみ更新） |
| **交差ポリゴン** | display_name は空文字、ID は自動生成 |
| **微小スライバー** | 面積が極小（< 1e-12）の結果は自動破棄 |
| **Undo** | 元ポリゴンの geometry を復元し、交差ポリゴンを削除 |
| **coordIndex** | 更新されたポリゴンは再インデックスされる |
| **最小入力** | 2つ以上の PolygonID が必要。1つだとエラー |
